================================================================================
  BERANS AUTO — как запустить проект (фронтенд + бэкенд)
================================================================================

Репозиторий: https://github.com/Pon4ikx/BernansAuto

Требования:
  • Python 3 (рекомендуется 3.10+)
  • Node.js и npm
  • PostgreSQL (для бэкенда)
  • Git

--------------------------------------------------------------------------------
  0. КЛОНИРОВАНИЕ РЕПОЗИТОРИЯ
--------------------------------------------------------------------------------

  Открыть терминал и перейти в папку, куда хотите сохранить проект:

    cd C:\Projects   (или другая удобная папка)

  Клонировать репозиторий:

    git clone https://github.com/Pon4ikx/BernansAuto.git

  Перейти в папку проекта:

    cd BernansAuto

  Структура проекта:

    BernansAuto/
      bernansauto/
        backend/     — Django-бэкенд (API, админка)
        frontend/    — React-фронтенд (интерфейс)
      Readme.txt
      commands.txt

--------------------------------------------------------------------------------
  1. БЭКЕНД (Django)
--------------------------------------------------------------------------------

1.1. Создать виртуальное окружение Python

    В корне проекта (BernensAuto):

    Windows (PowerShell или cmd):
      python -m venv .venv

    Linux / macOS:
      python3 -m venv .venv

1.2. Активировать виртуальное окружение

    Windows (PowerShell или cmd):
      .venv\Scripts\activate

    Linux / macOS:
      source .venv/bin/activate

    После активации в начале строки появится (.venv).

1.3. Установить зависимости Python

    Перейти в папку бэкенда:

      cd bernansauto\backend

    Установить все библиотеки из requirements.txt:

      pip install -r requirements.txt

    (Убедитесь, что venv активирован — pip установит пакеты в него.)

1.4. Настроить RAdmin и подключиться к сети с БД

    База данных не импортируется вручную — она уже доступна в сети.
    Скачать и запустить RAdmin:
      • Сеть — Подключиться к сети
      • Название сети: BernansAuto
      • Пароль: BernansAuto

    После подключения к сети можно подключаться к БД 


1.7. Запустить сервер бэкенда

      python manage.py runserver

    Бэкенд будет доступен по адресу:  http://127.0.0.1:8000/
    Админка Django:                   http://127.0.0.1:8000/admin/
    API (Swagger):                    http://127.0.0.1:8000/swagger/

--------------------------------------------------------------------------------
  2. ФРОНТЕНД (React)
--------------------------------------------------------------------------------

2.1. Установить зависимости Node.js

    Открыть второй терминал (venv для фронтенда не нужен).
    Перейти в папку фронтенда:

      cd bernansauto\frontend

    Установить все npm-пакеты:

      npm install

2.2. Запустить фронтенд

      npm start

    Откроется браузер с приложением по адресу:  http://localhost:3000

    Фронтенд по умолчанию обращается к бэкенду на порту 8000 (CORS уже настроен
    для http://localhost:3000 в settings.py бэкенда).

--------------------------------------------------------------------------------
  3. ИТОГО: как запускать каждый раз
--------------------------------------------------------------------------------

  Терминал 1 (бэкенд):
    cd BernansAuto\bernansauto\backend
    ..\..\.venv\Scripts\activate    (если venv в корне проекта)
    python manage.py runserver

  Терминал 2 (фронтенд):
    cd BernansAuto\bernansauto\frontend
    npm start

  В браузере открыть:  http://localhost:3000  — главная страница с каталогом
  и формой входа/регистрации.

--------------------------------------------------------------------------------
  4. КРАТКАЯ ШПАРГАЛКА (с нуля)
--------------------------------------------------------------------------------

  git clone https://github.com/Pon4ikx/BernansAuto.git
  cd BernansAuto

  python -m venv .venv
  .venv\Scripts\activate

  cd bernansauto\backend
  pip install -r requirements.txt
  python manage.py migrate
  python manage.py runserver

  (В другом терминале)
  cd bernansauto\frontend
  npm install
  npm start

--------------------------------------------------------------------------------
  5. Возможные проблемы
--------------------------------------------------------------------------------

  • "ModuleNotFoundError: No module named 'django'" — активируйте venv и
    установите зависимости (pip install -r requirements.txt).

  • Ошибка подключения к PostgreSQL — проверьте, что сервер PostgreSQL запущен,
    база BernansAuto_DATABASE создана, логин/пароль в settings.py совпадают.

  • Фронтенд не видит API — убедитесь, что бэкенд запущен на http://127.0.0.1:8000
    и в backend/bernansauto/settings.py в CORS_ALLOWED_ORIGINS есть "http://localhost:3000".

  • "python" не найден — установите Python и добавьте его в PATH, либо используйте
    py -m venv .venv (Windows).
